# Agent Instructions

このリポジトリで作業するエージェントは、必要に応じて `doc/` 以下のドキュメントを参照し、実装とドキュメントの差分が広がらないようにしてください。

## 作業前に参照するドキュメント

- データベース、Entity Framework Core、エンティティ、マイグレーション、テーブル構造に関わる変更を行う場合は、`doc/database-er.md` を確認してください。
- プロジェクト参照、レイヤ構成、DI 登録、主要な実装配置に関わる変更を行う場合は、`doc/project-dependencies.md` を確認してください。
- スコアリング、外部データ、判定ロジックに関わる変更を行う場合は、`doc/scoring.md` や `doc/external-data-scoring.md` を確認してください。

## 編集後のドキュメント更新

コードや設定の変更が `doc/` 以下の説明に影響する場合は、実装後に該当する Markdown を更新してください。

更新対象の例:

- DB スキーマ、エンティティ、リレーション、マイグレーションの変更
- プロジェクト参照、レイヤ責務、DI 登録、主要ディレクトリ構成の変更
- スコア計算、売買判断、外部データ利用方針、レポート内容の変更
- 環境変数、設定項目、外部サービス連携の変更

影響がない場合は、ドキュメント更新は不要です。判断に迷う場合は、関連する `doc/` を確認し、古くなる記述があれば最小限の更新を行ってください。

## 変更時の注意

- 既存のユーザー変更を巻き戻さないでください。
- ドキュメント更新は、実装差分に対応する範囲へ絞ってください。
- README と `doc/` の両方に関係する変更では、README の概要と `doc/` の詳細が矛盾しないようにしてください。

## 作業後の Git 操作

作業が完了したら、変更内容を確認したうえで `git commit` し、リモートへ `git push` してください。

- コミット前に `git status` と差分を確認してください。
- コミットには、実装変更とそれに対応するドキュメント更新を含めてください。
- 既存のユーザー変更や無関係な未追跡ファイルは、勝手にコミットへ含めないでください。
- push できない場合は、理由と未完了の操作をユーザーに伝えてください。
